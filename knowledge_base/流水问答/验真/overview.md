# 验真概述

## 项目背景与目标

在信贷审批、担保评估等业务中，银行流水是核实客户收入与经营状况的核心依据。目前市场上存在大量经过篡改或伪造的银行流水，传统的肉眼审核效率低且难以识别高仿真造假。

## 四种验真方式

| 验真方式 | 原理 | 适用账单类型 |
|---------|------|-------------|
| 结息验真 | 通过计算流水期间日均余额反推理论结息金额，与实际结息金额比对 | 银行卡 |
| 余额连续性验真 | 逐笔校验"上笔余额 ± 交易金额 = 本笔余额"，识别余额篡改或伪造 | 银行卡 |
| 交易日期验真 | 分析交易日期的分布规律，验证结息日期、交易频率是否符合真实账户行为特征 | 银行卡 |
| 电子签章 | 核查账单文件的数字签名和元数据是否来自官方渠道 | 支付宝/微信/银行卡 |

四种方式相互补充，从数学逻辑、统计规律、文件底层三个层面共同构建流水真实性的验证体系。

## 综合评分模型

### 有结息验真时

| 维度 | 权重 |
|------|------|
| 结息验真 | 40% |
| 余额连续性验真 | 35% |
| 交易日期验真 | 25% |

### 无结息验真时

| 维度 | 权重 |
|------|------|
| 余额连续性验真 | 55% |
| 交易日期验真 | 45% |

### 判定结论

| 综合评分 | 判定结论 | 说明 |
|---------|---------|------|
| 90~100 | 通过 | 各维度整体表现优秀，可信度高 |
| 70~89 | 预警 | 存在轻微异常，建议人工复核 |
| 0~69 | 拒绝 | 严重异常，疑似造假，原则上拒绝 |
